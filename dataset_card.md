# Benchmark Half-Life: Perturbation-Holdout Item Sets

## Dataset Description

Perturbation-holdout items for measuring data contamination in LLM benchmarks. Each item contains an original benchmark question paired with a semantically-equivalent perturbation (paraphrase for MCQ, number substitution for math, identifier rename for code). A model that has memorized the original surface form will score higher on the original than on the perturbation; the gap measures contamination.

### Benchmarks Covered

| Benchmark | Items | Perturbation Type |
|---|---|---|
| MMLU | 228 | GPT-4o-mini paraphrase |
| ARC-Challenge | 200 | GPT-4o-mini paraphrase |
| GSM8K | 200 | Number substitution |
| TruthfulQA | 200 | Claude haiku paraphrase |
| HumanEval | 164 | Docstring + test case rewrite |
| MBPP | 200 | Claude haiku paraphrase + identifier rename |

### Key Findings (from the paper)

- **GSM8K**: +13pp industry-wide contamination gap (identical across GPT and Claude vendors)
- **MMLU**: +1.6pp (mild, both vendors)
- **ARC**: +0.7pp (near-zero, ceiling-dominated)
- **TruthfulQA**: +1.0pp GPT, +5.2pp Claude (vendor-specific)
- **MBPP**: Paraphrase yields negative gap (method boundary); identifier rename yields near-zero (no surface-form memorization)

## Dataset Structure

Each JSON file contains a list of items with fields:
- `orig_question`, `orig_choices` (or `orig_code`), `answer`
- `pert_question`, `pert_choices` (or `pert_code`)
- Benchmark-specific metadata

## Intended Use

- Measuring contamination gaps on new LLM models
- Extending the perturbation-holdout protocol to additional benchmarks
- Reproducing the half-life analysis from the paper

## Limitations

- Perturbation quality varies by benchmark (see MBPP boundary discussion in paper)
- Items are a subsample, not the full benchmark
- Perturbations were generated by LLMs (GPT-4o-mini, Claude haiku) and may introduce subtle semantic shifts
